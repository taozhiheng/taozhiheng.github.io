Java Compile

Java Compile

总是在使用各种idea编译java项目，最近学习了下怎样自己编译java项目。主要时是以下命令的使用：

  • javac，编译
  • jar，打包
  • java，运行

注:下面的java命令时是linux下的，windows稍有不同

javac

  • 基本用法：
    javac -classpath dependency1:dependency2 -sourcepath src src/com/persist/Main.java -d classes
  • 参数说明：
    • -classpath dependency1:dependency2　编译依赖，可以是.jar, .zip, 目录，多个之间用:隔开，等价与(-cp)
    • -sourcepath src　源码路径
    • src/com/persist/Main.java　java程序主类入口
    • -d classes　编译后.class文件存放路径

jar

  • 基本用法：
    jar {ctxui}[vfm0Me] [jar-file] [manifest-file] [entry-point] [-C dir] file1 [-C dir] files …
  • 示例：
    jar cvf Download.jar Main -C classes com
  • 参数说明：
    • {ctxui}[vfmoMe]
      • -c　创建新的 JAR 文件包
      • -t　列出 JAR 文件包的内容列表
      • -x　展开 JAR 文件包的指定文件或者所有文件
      • -u　更新已存在的 JAR 文件包 (添加文件到 JAR 文件包中)
      • [vfm0M] 中的选项可以任选，也可以不选，它们是 jar 命令的选项参数
      • -v　生成详细报告并打印到标准输出
      • -f　指定 JAR 文件名，通常这个参数是必须的
      • -m　指定需要包含的 MANIFEST 清单文件
      • -e 指定程序主类入口
      • -0　只存储，不压缩，这样产生的 JAR 文件包会比不用该参数产生的体积大，但速度更快
      • -M　不产生所有项的清单（MANIFEST〕文件，此参数会忽略 -m 参数
      • -i 为制定指定的 JAR 文件生成索引信息
    • [jar-文件]　即需要生成、查看、更新或者解开的 JAR 文件包，它是 -f 参数的附属参数
    • [manifest-file]　Manifest文件
    • [entry-point]　程序主类入口
    • -C　转到指定的目录执行jar

java

  • 基本用法：
    java -Djava.ext.dirs=lib1/:lib2/ -classpath dependency1:dependency2 com.persist.Main argument1 argument2
  • 参数说明：
    • -Djava.ext.dirs=s=lib1/:lib2/　运行时外部依赖库目录，多个之间用:隔开（极端做法，将所有依赖放到java的lib目录下，或者将所有依赖打入jar包，就无需制定外部依赖）
    • -classpath dependency1:dependency2　运行时.class位置，可以是.jar, .zip, 目录，多个之间用:隔开，等价与(-cp)
    • com.persist.Main　java程序指定将要运行的类，含有main方法
    • argument1 argument2　main方法的参数

编写Makefile

工程结构如下：

  • —Download
    • —lib
      • —gson-2.7.jar
    • —src
      • —com
        • —persist
          • —Download.java
          • —Main.java
    • —Makefile
    • —classes

项目组织：

  • lib　建议放置所有第三方库在此目录下
  • src/com　建议最外层只有一个包目录，方便打包
  • classes　建议放置编译后的.class文件
